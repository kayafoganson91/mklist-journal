# mklist-journal

这是一个用于归档和发布独立 HTML 页面的仓库。仓库本身不绑定特定域名或网站，仅作为静态页面资源的集合与发布入口。

## 项目简介

mklist-journal 旨在提供一个轻量、可维护的 HTML 页面存档空间。您可以将各类独立页面（如信息页、工具页、展示页等）直接放置在仓库中，并通过 GitHub Pages 或其他静态托管服务进行访问。仓库结构简单，不依赖复杂框架，适合快速记录与分享。

## 目录说明

```
.
├── index.html          # 仓库根页面，可作为目录或入口
├── pages/              # 存放独立 HTML 页面的文件夹
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：每个 HTML 文件视为一个独立页面，可自由命名。
- `assets/`：存放页面共享的资源文件，建议按类型组织子目录。
- `index.html`：可自定义为导航页或说明页，便于访问者浏览。

## 页面归档说明

- 所有页面均为纯 HTML 文件，兼容主流浏览器。
- 页面之间彼此独立，不依赖外部数据库或后端服务。
- 资源引用建议使用相对路径，以确保在本地或托管环境中正常加载。
- 若需引用外部资源（如字体库、CDN 脚本），请注明来源并确保其可用性。

## 维护说明

- 新增页面：在 `pages/` 目录下创建 `.html` 文件，并更新 `index.html` 中的链接（如有需要）。
- 修改页面：直接编辑对应 HTML 文件，注意保持资源路径正确。
- 删除页面：移除文件并清理相关引用。
- 建议定期检查页面链接与资源是否失效，保持仓库整洁。

## 使用方式

1. 克隆或 fork 本仓库到您的 GitHub 账号下。
2. 按需修改或添加 HTML 页面。
3. 启用 GitHub Pages（Settings → Pages → 选择 main 分支，根目录或 `/docs` 目录）。
4. 访问生成的 Pages 链接即可查看页面。
